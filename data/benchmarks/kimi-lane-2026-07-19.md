## Findings

1. **83% of the Kimi child system prompt is the user skills index.** The
   dissected `_system_prompt` is 103,308 bytes; the `### User` skills section
   is 85,960 bytes. Pointing `--skills-dir` at an empty directory drops
   uncached input from ~22.9k to ~2.1k (**−91%**) and gross input from ~63.8k
   to ~22.0k (**−65%**) with the same oracle pass.
2. **Kimi bootstrap sits between Codex and Claude.** Lean child ≈ 10.8k gross
   first request vs the protocol's ~11k Codex assumption and the measured
   ~44k Claude child. Default (skills-loaded) child ≈ 31.8k first request.
3. **Moonshot caching is implicit and write-free.** `input_cache_creation`
   was 0 in every request; a stable prefix block (9,216 tokens on the default
   prompt) is served from cache across sessions without a write premium.
   Repeat runs still paid ~22.6k uncached — the prompt after that block is
   session-varying, so cross-session reuse is partial.

## Swarm A/B — simultaneous vs staggered (lean children)

Three disjoint slices (lines 1–1333, 1334–2666, 2667–4000); ground truth
40/32/28; controller sums deterministically. Both arms passed the oracle
(100 errors, marker 3777).

| Arm | Total uncached | Total gross input | Cache write | Output | Wall |
|---|---:|---:|---:|---:|---:|
| Simultaneous (3 at once) | 6,309 | 66,213 | 0 | 1,158 | 13.1 s |
| Staggered (1 then 2) | 6,345 | 66,249 | 0 | 978 | 21.7 s |

4. **Staggering buys nothing on Moonshot.** Per-child uncached input is
   identical across arms (~2.1k, noise-level delta); there is no cache-write
   premium to convert. Simultaneous fan-out is strictly better here — same
   tokens, 1.66x faster. The stagger rule in the subagent protocol is
   provider-specific to explicit-cache APIs (Anthropic-style), not universal.
5. **Kimi lean team vs measured Claude team:** 66,213 gross input vs 390,010
   ([claude-team-ab-2026-07-16](claude-team-ab-2026-07-16.md)) — **17%** of
   the Claude team's gross for the same oracle shape on this host.
